# Official 1.7.7 合并设计

## 背景

`official` 分支已经更新到提交 `b0e21ff`，对应官方油猴脚本版本 `1.7.7`。本仓库 `main` 分支是非官方二开版本：它保留了可测试的 Node/Jest 工程结构，并新增了 GPA 预测功能。

这次合并的目标不是让官方脚本覆盖本仓库，而是把 official `1.7.7` 的有效行为移植到 `main`，同时保留 GPA 预测、README、测试体系和开发流程。

当前确认的基线：

- `main`：`34aa506`，提交信息为 `refactor: improve routing and test coverage`
- `official`：`b0e21ff`，提交信息为 `翱翔教务功能加强v1.7.7`
- 当前 `main` 测试基线：执行 `npm test -- --runInBand` 时，`8` 个测试套件、`115` 个测试全部通过

## 目标

- 将 official `v1.7.7` 的核心意图合并进 `main`。
- 保留非官方二开身份。
- 保留 GPA 预测，以及它依赖的课表缓存流程。
- 保留 Node/Jest 自动化测试和现有开发工作流。
- 只做服务于合并、可读性和可测试性的必要重构。
- 对行为变更使用测试驱动开发。

## 非目标

- 不用 official 的单文件脚本整体替换 `main.user.js`。
- 不删除 `README.md`、`package.json`、`package-lock.json` 或测试文件。
- 不重新设计 GPA 预测 UI。
- 不加入与本次合并无关的新功能。
- 除非后续实施计划单独说明，不在本次合并中把油猴脚本拆成构建期多模块工程。

## Official 1.7.7 的意图

过滤空白和格式噪音后，official `v1.7.7` 的有效变化主要集中在以下行为：

- 将脚本元信息版本从 `1.7.6` 更新到 `1.7.7`。
- 增强全校开课查询页的同步功能，增加可选学期下拉框。
- 手动同步时，只抓取用户选中的学期。
- 自动同步时，按学期选项寻找最新且有数据的学期。
- 将包含 `无数据` 的 `td.dataTables_empty` 视为空学期，而不是继续等待加载。
- 抓取课程行时跳过 DataTables 的空行。
- 后台同步使用同样的“寻找最新有数据学期”逻辑。
- 教师主页自动搜索等待输入框和按钮都出现后再提交，并在真正点击搜索后再清空待搜索姓名。

## 必须保留的 main 行为

`main` 分支已有的二开能力和工程化改动必须保留：

- 油猴脚本名称保持 `翱翔教务功能加强(非官方)`。
- 油猴脚本版本更新为 `1.7.7`。
- 悬浮菜单保留 `GPA预测` 入口。
- 保留 `CONSTANTS.GPA_ESTIMATE_KEY` 和课表缓存相关常量。
- 保留 `handleGpaEstimateClickImmediate`、`handleGpaEstimateClickLoad`、`calculatePredictedGPA`、`formatCacheAge` 等 GPA 预测入口。
- 保留课表自动跳转、自动缓存和课程学分提取逻辑，因为 GPA 预测依赖当前学期课程学分。
- 保留测试模式启动保护和 `globalThis.__NWPU_EDU_PLUS_TEST_EXPORTS`。
- 保留更新下载 fallback 辅助函数。
- README 和 package 配置继续说明本仓库是非官方二开，并保留开发命令。

## 架构方案

以 `main` 为基底，定向移植 official `v1.7.7` 的有效行为。

这样可以保留现有测试体系、路由拆分、GPA 预测代码和可测试入口。实现时避免整段覆盖 `main.user.js`，而是对每个 official 行为先写或更新测试，再做最小生产代码改动，最后在绿灯后整理命名和结构。

## 组件设计

### 脚本元信息

只更新版本字段：

- `@name` 保持 `翱翔教务功能加强(非官方)`。
- `@version` 更新为 `1.7.7`。

### 全校开课查询同步

更新 `LessonSearchEnhancer`，让控制面板包含学期下拉框和“存储选定学期课程信息”按钮。

模块需要做到：

- 优先从页面 selectize 数据中填充 `#gm-sync-semester`。
- selectize 数据不可用时，从 DOM 下拉选项中兜底提取。
- 保留当前历史余量标签渲染能力。
- 手动模式要求有选定学期；如当前页面不是该学期，则切换到该学期后再抓取。
- 手动模式遇到空学期时，给出提示并移除遮罩。
- 自动模式按最新到最旧扫描学期，选择第一个有数据的学期。
- 同步遮罩显示最终锁定的学期。

### 后台同步

更新 `BackgroundSyncSystem`，让后台 worker 在抓取前寻找最新有数据的学期。

后台同步需要做到：

- 打开页面原有学期下拉框。
- 逐个尝试学期选项。
- 每次切换后等待加载完成。
- 遇到包含 `无数据` 的表格时跳过该学期。
- 如果所有学期都没有数据，抛出清晰错误。
- 抓取行时忽略 DataTables 的空行。

### 教师主页搜索

保留 `main` 已经抽出的教师搜索辅助函数，并把 official 修复合入这些函数。

行为需要做到：

- 等待 `#sea` 输入框和 `.dyym2_btn` 搜索按钮都出现。
- 把待搜索姓名写入输入框。
- 点击搜索前派发 `input` 和 `change` 事件，适配页面框架监听。
- 只有真正点击搜索按钮后，才清空 `gm_cross_search_name`。
- `teacher.nwpu.edu.cn` 的非搜索页不能落入 JWXT 首页初始化逻辑。

### GPA 预测

GPA 预测是本仓库二开功能，应作为 fork-owned 功能保留。

行为继续保持：

- 悬浮菜单中存在 `GPA预测` 按钮。
- 预测弹窗读取已缓存成绩和课表缓存。
- 未出分课程支持选择或输入预估 GPA。
- 未填写预估 GPA 时显示用户可理解的提醒。
- 结果显示本学期 GPA 和预测总 GPA。
- 预估值继续按课程代码存储在 `GPA_ESTIMATE_KEY` 下。

## 数据流

全校开课查询同步：

1. 控制面板等待表格和分页控件出现后初始化。
2. 学期下拉框从页面现有学期选项中填充。
3. 手动同步读取用户选中的学期；自动同步按最新到最旧扫描学期。
4. 页面通过原有 selectize UI 切换学期。
5. 抓取逻辑等待加载完成，或识别空学期标记。
6. 有效课程行转换为历史记录，写入 `HISTORY_STORAGE_KEY`。

GPA 预测：

1. 用户从悬浮菜单打开 GPA 预测。
2. 弹窗读取成绩缓存和 `COURSE_TABLE_CACHE_KEY`。
3. 当前学期未出分课程读取 `GPA_ESTIMATE_KEY` 中保存的预估值。
4. 计算逻辑合并历史成绩、当前学期已出分课程和当前学期预估课程。
5. 弹窗显示本学期 GPA 和预测总 GPA。

教师主页搜索：

1. 待搜索姓名写入 `gm_cross_search_name`。
2. 教师搜索页轮询等待输入框和搜索按钮。
3. 输入姓名并提交搜索。
4. 搜索按钮被真正点击后清空队列。

## 错误处理

- 学期列表加载失败时，在下拉框中显示加载失败，并停止同步。
- 手动选择的学期无数据时，提示用户并移除同步遮罩。
- 自动同步找不到任何有数据学期时，前台流程提示用户，后台流程抛出清晰错误。
- 后台同步遇到 DataTables 空行时跳过，避免生成无效课程记录。
- 教师搜索超时时，应与辅助函数测试保持一致；首选行为是没有成功提交时保留待搜索姓名，便于刷新后重试。
- GPA 预测缺少课程学分时，保留现有提示，引导用户先缓存课表。

## 测试策略

每个行为切片都使用 TDD：

- 为脚本元信息增加或更新测试，确认非官方名称和 `1.7.7` 版本。
- 为全校开课查询同步增加测试，覆盖学期下拉框渲染和选定学期同步。
- 为自动同步增加测试，覆盖跳过空学期并锁定有数据学期。
- 为后台同步增加测试，覆盖跳过空行和跳过空学期。
- 为教师搜索增加测试，覆盖事件派发和成功提交后清空队列。
- 运行现有 GPA 预测测试，确认二开功能仍然工作。
- 完成后运行语法检查和全量测试。

必须执行的验证命令：

```bash
node --check main.user.js
npm test -- --runInBand
```

## 开发流程

每个实现任务遵循红灯、绿灯、重构：

1. 先写失败测试。
2. 运行聚焦测试，确认它因为预期原因失败。
3. 写最小生产代码让测试通过。
4. 再运行聚焦测试，确认通过。
5. 相关任务完成后运行全量测试。
6. 提交保持小而聚焦。
